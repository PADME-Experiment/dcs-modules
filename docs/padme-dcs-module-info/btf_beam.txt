BTFDATA data specification

Luca Foggetta has prepared 2 keys for the data relevant for PADME data taking onto the on the memcached server 192.168.198.164.
The 2 keys are :
VUG_PADME
BTFDATA_PADME

Follow the instructions from Luca :

Each one has more field, comma separated, in ASCII char human readable format, having this structure (one item in sequence):
1) copy of memcached key name in uppercase
2) timestamp from 12:00AM, 1st Jan 1904, in second.usecond format
3) a logic ASCII bool (0,1): if zero the producer machine work correctly, if one the data are not useful
4) producer machine data, comma separated

-----------------------------------------------------------------

For VUG_PADME, the 4) machine data are  :
    a,sx.xxxxEsxx<CR><LF><CR><LF>

a = Description Status,
0 –>  Measurement data okay
1 –>  Underrange
2 –>  Overrange
3 –>  Sensor error
4 –>  Sensor off (IKR, PKR, IMR, PBR)
5 –>  No sensor (output: 5,2.0000E-2 [hPa])
6 –> Identification error


sx.xxxxEsxx = Measurement value
 [in current pressure unit]
(s = sign)
Values always in exponential format


---------------------------------

For BTFDATA_PADME, the 4) machine data are:
a,b,c,d,e,f,g,h<CR><LF><CR><LF>

a=logic ASCII bool (0,1)= DHPTB101 ok and on
b=logic ASCII bool (0,1)= BTF particle type 1=positron, 0=electron
c=logic ASCII bool (0,1)= DHSTB001 ok (magnet on and current set reached)
d=logic ASCII bool (0,1)= DHSTB002 ok (magnet on and current set reached)
e=logic ASCII bool (0,1)= DHRTB102 ( = Padme magnet) ok (magnet on and current set reached)
f=floating point ASCII value = DHSTB001 steering equivalent electron energy (ideal particle) [MeV]
g=floating point ASCII value = DHSTB002 steering equivalent electron energy (ideal particle) [MeV]
h=floating point ASCII value = DHRTB102 (=Padme magnet) current set

-------------------------------------------------------------------------------------------------------

The DrvBTFBeam module reads from the memcached server these values, transforms the Labview timestamp into local CET timestamp
and sends data in the following form (separator is always ; character)

BFTDATA      --> values are written and updated every 2 sec  into the padme-dcs/data/BTFDATA_Padme   file
Timestamp(CET) [string] ; producer status [decimal] ; DHPTB101 status [decimal] ; BTF particle type [decimal] ; DSHTB001 status [decimal] ;
 DHSTB02 status [decimal] ; Padme magnet status [decimal]; energy from DHSTB001 [float] ; energy from DHSTB002 [decimal] ; Padme magnet current in A [decimal]

example:
Wed Oct 10 12:19:48 2018;0;1;1;1;1;1;545.15;542.33;283.00 

 
VUGDATA      --> values are written and updated every 2 sec  into the padme-dcs/data/VUG_Padme   file
Timestamp(CET) [string] ;  producer status [decimal] ;  readout device status [decimal] ; pressure in mbar [scientific notation]

example:
Wed Oct 10 12:23:53 2018;0;0;+7.8100E-07
